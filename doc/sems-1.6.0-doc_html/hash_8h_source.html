00004 /*
00005 -------------------------------------------------------------------------------
00006 lookup3.c, by Bob Jenkins, May 2006, Public Domain.
00007 
00008 These are functions for producing 32-bit hashes for hash table lookup.
00009 hashword(), hashlittle(), hashlittle2(), hashbig(), mix(), and final() 
00010 are externally useful functions.  Routines to test the hash are included 
00011 if SELF_TEST is defined.  You can use this free for any purpose.  It's in
00012 the public domain.  It has no warranty.
00013 
00014 You probably want to use hashlittle().  hashlittle() and hashbig()
00015 hash byte arrays.  hashlittle() is is faster than hashbig() on
00016 little-endian machines.  Intel and AMD are little-endian machines.
00017 On second thought, you probably want hashlittle2(), which is identical to
00018 hashlittle() except it returns two 32-bit hashes for the price of one.  
00019 You could implement hashbig2() if you wanted but I haven't bothered here.
00020 
00021 If you want to find a hash of, say, exactly 7 integers, do
00022   a = i1;  b = i2;  c = i3;
00023   mix(a,b,c);
00024   a += i4; b += i5; c += i6;
00025   mix(a,b,c);
00026   a += i7;
00027   final(a,b,c);
00028 then use c as the hash value.  If you have a variable length array of
00029 4-byte integers to hash, use hashword().  If you have a byte array (like
00030 a character string), use hashlittle().  If you have several byte arrays, or
00031 a mix of things, see the comments above hashlittle().  
00032 
00033 Why is this so big?  I read 12 bytes at a time into 3 4-byte integers, 
00034 then mix those integers.  This is fast (you can do a lot more thorough
00035 mixing with 12*3 instructions on 3 integers than you can with 3 instructions
00036 on 1 byte), but shoehorning those bytes into integers efficiently is messy.
00037 -------------------------------------------------------------------------------
00038 */

00043 /*
00044 --------------------------------------------------------------------
00045  This works on all machines.  To be useful, it requires
00046  -- that the key be an array of uint32_t's, and
00047  -- that the length be the number of uint32_t's in the key
00048 
00049  The function hashword() is identical to hashlittle() on little-endian
00050  machines, and identical to hashbig() on big-endian machines,
00051  except that the length has to be measured in uint32_ts rather than in
00052  bytes.  hashlittle() is more complicated than hashword() only because
00053  hashlittle() has to dance around fitting the key bytes into registers.
00054 --------------------------------------------------------------------
00055 */

00075 /*
00076 -------------------------------------------------------------------------------
00077 hashlittle() -- hash a variable-length key into a 32-bit value
00078   k       : the key (the unaligned variable-length array of bytes)
00079   length  : the length of the key, counting by bytes
00080   initval : can be any 4-byte value
00081 Returns a 32-bit value.  Every bit of the key affects every bit of
00082 the return value.  Two keys differing by one or two bits will have
00083 totally different hash values.
00084 
00085 The best hash table sizes are powers of 2.  There is no need to do
00086 mod a prime (mod is sooo slow!).  If you need less than 32 bits,
00087 use a bitmask.  For example, if you need only 10 bits, do
00088   h = (h & hashmask(10));
00089 In which case, the hash table should have hashsize(10) elements.
00090 
00091 If you are hashing n strings (uint8_t **)k, do it like this:
00092   for (i=0, h=0; i<n; ++i) h = hashlittle( k[i], len[i], h);
00093 
00094 By Bob Jenkins, 2006.  bob_jenkins@burtleburtle.net.  You may use this
00095 code any way you wish, private, educational, or commercial.  It's free.
00096 
00097 Use for hash table lookup, or anything where one collision in 2^^32 is
00098 acceptable.  Do NOT use for cryptographic purposes.
00099 -------------------------------------------------------------------------------
00100 */
00101 
00102 uint32_t hashlittle( const void *key, size_t length, uint32_t initval);
